# Contributing to VisOnboarding

Contributions are welcome. If you're unsure whether something is a good fit, open an issue to discuss it first.

## How to Contribute

1. Fork the repository.
2. Create a feature branch (`git checkout -b my-change`).
3. Make your changes and keep them consistent with the existing code style.
4. Verify locally:
   - `npm install`
   - `npm run build`
   - (optional) `npm run dev`
5. Open a pull request describing what you changed and why.

## Environment Variables

Do not commit secrets. Use `.env.example` as the template for required environment variables.

## Security

If you believe you've found a security issue, please open an issue describing the problem and (if possible) how to reproduce it.
